# 🎬 Terminal Demo

A real walkthrough of the API via `curl` — actual requests and responses captured against the
running stack. Start the backing services with `docker compose up -d` and the app with
`./mvnw spring-boot:run`, then follow along.

> Owner IDs below are the JWT subject (the Keycloak user's UUID); they're shown truncated for readability.

## 1. Get an access token (Keycloak)

```bash
TOKEN=$(curl -s -X POST http://localhost:8082/realms/myrealm/protocol/openid-connect/token \
  -d grant_type=password -d client_id=springboot-app -d client_secret=secret123 \
  -d username=user1 -d password=userpass | jq -r .access_token)
```

## 2. Create tasks

```bash
curl -s -X POST http://localhost:8080/api/v1/tasks \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"title":"Prepare Q3 report","description":"draft the quarterly financials","completed":false}'
```
```json
{
  "id": 35,
  "title": "Prepare Q3 report",
  "description": "draft the quarterly financials",
  "completed": false,
  "owner": "9050e08b…",
  "createAt": "2026-07-05T10:17:14.47",
  "updatedAt": "2026-07-05T10:17:14.47",
  "version": 0
}
```

## 3. List your tasks (paginated, owner-scoped)

```bash
curl -s "http://localhost:8080/api/v1/tasks?size=3" -H "Authorization: Bearer $TOKEN"
```

## 4. Full-text search (Elasticsearch, highlighted, owner/tenant-scoped)

Tasks are indexed **event-driven** (write → outbox → Kafka → search-index consumer → Elasticsearch),
so a new task becomes searchable a moment after it's created.

```bash
curl -s "http://localhost:8080/api/v1/tasks/search?q=report" -H "Authorization: Bearer $TOKEN"
```
```json
{
  "results": [
    {
      "id": 35,
      "title": "Prepare Q3 report",
      "description": "draft the quarterly financials",
      "completed": false,
      "highlights": ["Prepare Q3 <em>report</em>"]

    {

      "title": "Book venue for the report review",
      "description": "reserve the conference room",
      "completed": false,
      "highlights": ["Book venue for the <em>report</em> review"]
    }

}
```

## 5. AI assistant (RAG over your tasks)

Retrieval (pgvector) finds your relevant tasks; the answer is generated by the LLM. The response below
is the **graceful fallback** returned when no `OPENAI_API_KEY` is set — note that retrieval still worked
(`retrievedTaskIds`), and the LLM call degraded via Resilience4j instead of failing. With a real key,
`answer` contains a generated response.

```bash
curl -s -X POST http://localhost:8080/api/v1/assistant/query \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"question":"What do I need to prepare for Q3?"}'
```
```json
{
  "answer": "The assistant is temporarily unavailable. Here are the tasks that matched your question.",
  "retrievedTaskIds": [35, 36, 37, 1, 2]
}
```
